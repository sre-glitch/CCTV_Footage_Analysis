## Logical roles (`store_layout.json`)

| Logical ID | Layout zones | Pipeline behavior |
|------------|--------------|-----------------|
| `ENTRY_CAMERA` | ENTRY, EXIT, FOH | Emits `ENTRY`, `EXIT`, `REENTRY`; creates visitor sessions |
| `FLOOR_CAMERA` | FOH, MAKEUP_UNIT, lower aisles | Zone telemetry on active session |
| `BILLING_CAMERA` | CASH_COUNTER | Queue join/abandon + billing zone presence for POS correlation |
| `AUX_CAMERA_1` | Upper brand grid | Zone telemetry only |
| `AUX_CAMERA_2` | MENS_CARE … BEAUTY | Zone telemetry only |

## Cross-camera rules

- Only `ENTRY_CAMERA` increments unique visitors.
- Other cameras attach `ZONE_*` / queue events to the active entry session.
- Re-run pipeline after any `camera_map.json` change.

## Review workflow

```bash
pnpm pipeline:preview
# Open data/previews/*_detections.jpg and data/previews/camera_preview_manifest.json
pnpm pipeline:cctv
pnpm db:seed:events
```
